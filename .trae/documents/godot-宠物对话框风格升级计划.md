# Godot 宠物对话框风格升级计划

## 目标

本轮只改快捷对话窗口，不动主应用窗口和桌宠主窗口的核心交互逻辑。目标包含三项：

1. 取消“打开应用”和“关闭”按钮
2. 取消左上角现有文字标题，改为“宠物圆形头像 + 名称”头部
3. 将整体风格继续升级成更像游戏里“与宠物对话”的浮窗样式

同时保持以下能力不回退：

- 发送输入仍可正常触发 `quick_task_requested`
- 状态文案仍可更新
- 快捷对话窗口仍可拖拽
- 现有透明玻璃风基础仍保留，但视觉重心转向“宠物对话框”而非“工具面板”

## 当前现状

基于当前工程，快捷对话窗口由以下文件构成：

- 场景：`godot-app/scenes/pet/QuickChatPanel.tscn`
- 逻辑：`godot-app/scripts/pet/QuickChatPanel.gd`

当前头部结构是：

- 左侧标题文字“简洁对话框”
- 下方状态文字
- 右侧 `QQPAL` 小徽标

当前底部操作区是：

- 输入框
- 发送按钮
- 打开应用按钮
- 关闭按钮

当前拖拽逻辑绑定在：

- `HeaderRow` 的 `gui_input`

这意味着本轮的重点是**重构头部视觉结构与底部操作区**，而不是重写发送逻辑。

## 设计方向

### 一、从“工具面板”切换到“宠物对话框”

这轮不继续强调“设置感”，而改成更像游戏中和宠物对话的浮窗体验：

- 头部像宠物身份卡
- 主内容像对话气泡
- 输入区简洁集中
- 控件更少、更像陪伴式交互

### 二、取消两个次要按钮

本轮直接移除：

- `打开应用`
- `关闭`

原因：

- 它们让快捷对话框更像工具面板
- “打开应用”可以保留为桌宠右键菜单入口，不一定要继续占用快捷对话面积
- “关闭”可以改为：
  - 点击窗口外关闭（后续再做），或
  - 使用系统关闭行为，或
  - 保留 ESC/失焦策略（后续再补）

首轮先完成视觉收敛，不同时引入新的关闭交互复杂度。

### 三、头部改成宠物身份区

左上角现有标题文本不保留，改成：

- 左侧圆形宠物头像
- 右侧两行信息：
  - 宠物名称
  - 当前状态文案

头部右上角不再放当前 `QQPAL` 小徽标的独立文本样式，而是让“名称”直接成为主体。

## 实施步骤

### 第一步：重构头部节点结构

目标：从“标题栏”变成“宠物身份栏”。

实施内容：

1. 删除现有头部里的：
   - 标题“简洁对话框”
   - 右侧 `QQPAL` 徽标式文本
2. 新增头像容器，头部改成：
   - 圆形头像
   - 名称
   - 状态文案
3. 头像优先采用以下方案之一：
   - 直接复用当前项目已有图标资源做第一版头像
   - 或使用桌宠首帧缩略图作为头像来源
4. 保留 `HeaderRow` 作为拖拽命中区域，不改变当前拖拽逻辑入口

验收结果：

- 顶部一眼看上去像“宠物身份区”
- 不再像旧式工具窗口标题

### 第二步：移除底部“打开应用 / 关闭”按钮

目标：简化对话框，只保留最核心输入行为。

实施内容：

1. 从场景中删除：
   - `OpenAppButton`
   - `CloseButton`
2. 删除对应连接：
   - `_on_open_app_button_pressed`
   - `_on_close_button_pressed`
3. 处理脚本中不再需要的信号与逻辑：
   - 评估是否移除 `open_app_requested`
   - 评估是否移除 `close_requested`
4. 保证 `SendButton` 和输入框逻辑不受影响

验收结果：

- 底部只保留输入行为
- 视觉更像即时对话框而不是控制面板

### 第三步：将面板改成“宠物对话气泡”风格

目标：让窗口视觉从玻璃卡片进一步靠近“游戏宠物对话框”。

实施内容：

1. 调整整体面板样式：
   - 保留半透明感
   - 减弱纯工具面板感
   - 提升宠物对话感
2. 风格重点包括：
   - 更柔和的圆角
   - 更温暖的配色
   - 头像区与消息区分层
   - 更像“角色说话框”
3. 优先保留当前 StyleBox 体系，但调整：
   - 面板主背景
   - 描边颜色
   - 阴影强度
   - 消息区样式
   - 输入区样式
4. 如果版面允许，增加轻量“气泡感”：
   - 通过边距、分区、卡片层次实现
   - 首轮不强求复杂尾巴形状，避免实现成本过高

验收结果：

- 视觉上更像和宠物对话
- 少一些“系统工具感”

### 第四步：为头像建立可复用资源入口

目标：让头部头像不是一次性写死。

实施内容：

1. 评估当前可直接复用的资源：
   - `icon.svg`
   - 桌宠 idle 首帧
2. 先确定第一版头像来源
3. 在场景中预留后续替换正式头像资源的结构
4. 保证头像显示为圆形或近似圆形视觉效果

验收结果：

- 头像不是临时占位文字
- 后续替换正式宠物头像不需要重构布局

### 第五步：保留拖拽能力但避免交互误伤

目标：在头部视觉变化后，拖拽仍然稳定。

实施内容：

1. 保留 `HeaderRow` 的拖拽逻辑
2. 调整头部结构时确保：
   - 头像区不会阻断拖拽
   - 名称与状态文本不会抢占事件
3. 若新结构导致拖拽命中缩小，则适度扩大 `HeaderRow` 有效区域

验收结果：

- 新头部仍然可拖动窗口
- 不会因为头像和文本重构导致拖拽失效

### 第六步：功能回归检查

目标：确保只是视觉升级，没有引入行为回退。

检查项：

1. 输入框可正常输入
2. 发送按钮可正常触发消息发送
3. 状态文本仍能通过 `set_status_text()` 更新
4. 头部拖拽仍然有效
5. 窗口透明与玻璃效果不丢失

## 预计改动文件

本轮预计主要修改：

- `godot-app/scenes/pet/QuickChatPanel.tscn`
- `godot-app/scripts/pet/QuickChatPanel.gd`

根据头像资源接入方式，可能补充调整：

- `godot-app/scripts/Main.gd`

如果使用现有资源作为头像，一般不需要额外新增脚本。

## 风险与处理

### 风险 1：移除“打开应用 / 关闭”后，快捷对话失去显式出口

处理方式：

- 第一轮先按用户要求移除
- 后续如需要，再补：
  - 失焦自动隐藏
  - ESC 关闭
  - 小型关闭图标

### 风险 2：头像素材不够精致

处理方式：

- 第一轮先用现有资源搭结构
- 重点先让“布局和风格方向”正确
- 等正式头像资源到位后再替换

### 风险 3：气泡感做得过头影响输入区可读性

处理方式：

- 首轮优先做“温暖玻璃 + 宠物身份区”
- 不强行加入复杂尾巴或过多装饰

## 最终交付预期

改完后，快捷对话框应满足：

1. 不再出现“打开应用”和“关闭”按钮
2. 左上角旧标题被移除
3. 顶部改成“宠物圆形头像 + 名称 + 状态”结构
4. 整体观感更像游戏里和宠物对话的浮窗
5. 拖拽与发送功能保持正常
